🏗️ Architecture Overview

Every package folder is named to tell you exactly which library and provider it contains.

com.example │ ├── agent/ ← Spring AI + Anthropic Claude (PRIMARY — 9 controllers) │ ├── config/ OpenAPI, Swagger + CorsConfig (allowedOriginPatterns("*")) │ ├── model/ 4 typed Java records for structured output │ ├── tools/ 15 @Tool functions the AI can call │ ├── service/ 7 services (agent, streaming, structured, templates...) │ └── controller/ 9 REST controllers │ ├── springai/ ← Spring AI — same API, 3 additional providers │ ├── openai/ OpenAI GPT-4o-mini → /api/springai/openai/* │ ├── ollama/ Ollama (local, free) → /api/springai/ollama/* │ └── mistral/ Mistral AI → /api/springai/mistral/* │ └── langchain4j/ ← LangChain4j — alternative framework, 5 providers ├── openai/ OpenAI → /api/langchain4j/openai/* ├── anthropic/ Anthropic Claude → /api/langchain4j/anthropic/* ├── ollama/ Ollama (local) → /api/langchain4j/ollama/* ├── gemini/ Google Gemini → /api/langchain4j/gemini/* └── mistral/ Mistral AI → /api/langchain4j/mistral/* │ └── resources/static/ ← Spring Boot auto-serves as http://localhost:8080/ └── index.html Interactive SPA playground — 19 pages, 73 live endpoints
💡
Why multiple providers coexist without conflicts: Spring AI creates typed beans per provider (e.g. OpenAiChatModel, AnthropicChatModel). LangChain4j uses explicit @Qualifier names (lc4jOpenAiChatModel, etc.). Services inject specific types — no @Primary needed.

⛓️ LangChain4j — Alternative Framework

Fundamentally different API — callback-based streaming, AiServices interfaces, annotation-driven prompts. Five providers.

💡
LangChain4j key concepts: ChatLanguageModel.generate() for direct calls · AiServices.builder() for interface proxies · @SystemMessage on methods · @V("name") for template variables · @MemoryId for session routing
java — LangChain4j AiServices pattern (same across all 5 providers)
@FunctionalInterface
interface Assistant {
    @SystemMessage("You are an expert in {domain}. Use {format} format.")
    String chat(@V("domain") String domain, @V("format") String format, @UserMessage String message);
}

@FunctionalInterface
interface MemoryAssistant {
    String chat(@MemoryId String sessionId, @UserMessage String message); // routes to per-session memory
}

// Build once, reuse across requests
var memoryAssistant = AiServices.builder(MemoryAssistant.class)
    .chatLanguageModel(chatModel)
    .chatMemoryProvider(id -> MessageWindowChatMemory.withMaxMessages(10))
    .build();

java — LangChain4j streaming bridged to Flux<String>
public Flux<String> streamChat(String message) {
    return Flux.create(sink ->
        streamingChatModel.generate(message, new StreamingResponseHandler<AiMessage>() {
            @Override public void onNext(String token)                { sink.next(token); }
            @Override public void onComplete(Response<AiMessage> r)  { sink.complete(); }
            @Override public void onError(Throwable error)           { sink.error(error); }
        }));
}

CorsConfig.java — New
Supports opening index.html from disk
java — src/main/java/…/config/CorsConfig.java
@Configuration
public class CorsConfig implements WebMvcConfigurer {
  @Override
  public void addCorsMappings(CorsRegistry registry) {
    registry.addMapping("/api/**")
      .allowedOriginPatterns("*")  // allows null (file://)
      .allowedMethods("GET","POST","DELETE","OPTIONS")
      .allowCredentials(false);
  }
}
💡
Without allowedOriginPatterns("*"), browsers block requests from file:// (null origin). This config enables the playground to work even when opened directly from disk.
